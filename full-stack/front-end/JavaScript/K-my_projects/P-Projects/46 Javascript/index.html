JavaScript Basics

Variables: var, let, and const are used to declare variables in JavaScript.

var a = 10;
let b = 20;
const c = 30;

Output: a = 10, b = 20, c = 30

Data Types: Primitive types include String, Number, Boolean, Undefined, Null.

const str = "Hello";
const num = 42;
const bool = true;

Output: str = "Hello", num = 42, bool = true

Operators: Arithmetic, Comparison, Logical.

const sum = 5 + 3;
const isEqual = 5 == "5";
const logical = true && false;

Output: sum = 8, isEqual = true, logical = false

Control Flow: Conditional and Looping constructs (if, switch, for, while).

if (a > b) {
console.log("a is greater");
} else {
console.log("b is greater");
}

Output: "b is greater"

Advanced JavaScript

Closures: Functions that remember their lexical scope.

function outer() {
let outerVar = "I'm outside!";
function inner() {
console.log(outerVar);
}
return inner;
} const innerFunc = outer();
innerFunc();

Output: "I'm outside!"

Promises: For handling asynchronous operations.

const myPromise = new Promise((resolve, reject) => {
let success = true;
if (success) {
resolve("Operation successful!");
} else {
reject("Operation failed!");
} }); myPromise.then(res => console.log(res));

Output: "Operation successful!"

Asynchronous Functions: Using async/await for cleaner code.

async function fetchData() {
let data = await fetch('https://api.example.com');
console.log(data); } fetchData();

Output: [Object with API data]

DOM Manipulation

Selecting Elements: Using document.getElementById(), querySelector().

const element = document.getElementById('myId');
const elements = document.querySelectorAll('.myClass');

Output: Selected elements based on ID and class.

Changing Content: Using innerHTML, textContent.

element.innerHTML = "New Content";
element.textContent = "Just Text";

Output: Updated content of the element.

Event Listeners: Adding click, hover events.

element.addEventListener('click', () => {
alert("Element clicked!");
});

Output: Alert on click event.

Object-Oriented Programming

Classes and Objects: Creating classes in JavaScript.

class Car {
constructor(brand) {
this.brand = brand;
}
displayBrand() {
console.log(this.brand);
}
} const myCar = new Car('Toyota');
myCar.displayBrand();

Output: "Toyota"

Inheritance: Extending classes.

class ElectricCar extends Car {
constructor(brand, battery) {
super(brand);
this.battery = battery;
}
displayBattery() {
console.log(this.battery);
}
} const myElectricCar = new ElectricCar('Tesla', '100 kWh');
myElectricCar.displayBattery();

Output: "100 kWh"

Asynchronous JavaScript

Callbacks: Using callbacks for asynchronous operations.

function fetchData(callback) {
setTimeout(() => {
callback("Data fetched!");
}, 1000); } fetchData(data => console.log(data));

Output: "Data fetched!" after 1 second.

ES6+ Features

Arrow Functions: Shorter syntax for functions.

const add = (a, b) => a + b;
console.log(add(5, 3));

Output: 8

Template Literals: Multi-line strings and string interpolation.

const name = "John";
const greeting = `Hello, ${name}!`;
console.log(greeting);

Output: "Hello, John!"

Error Handling

Try...Catch: Handling errors gracefully.

try {
let result = riskyFunction();
} catch (error) {
console.error("Error occurred:", error);
}

Output: Error message if riskyFunction fails.

Design Patterns

Module Pattern: Encapsulation of variables and functions.

const Module = (() => {
let privateVar = "I'm private";
return {
getPrivateVar: () => privateVar
}
})(); console.log(Module.getPrivateVar());

Output: "I'm private"

JavaScript APIs

Fetch API: For making HTTP requests.

fetch('https://api.example.com/data')
.then(response => response.json())
.then(data => console.log(data));

Output: Data from the API.

Event-Driven Programming

Event Listeners: Listening for user interactions.

document.addEventListener('click', () => {
console.log("Document clicked!");
});

Output: Log message when the document is clicked.

Data Structures

Arrays: List-like objects.

const fruits = ['Apple', 'Banana', 'Cherry'];
console.log(fruits[0]);

Output: "Apple"

Objects: Key-value pairs.

const car = {
brand: 'Toyota',
model: 'Corolla'
}; console.log(car.brand);

Output: "Toyota"

Unit Testing

Jest: A popular testing framework.

test('adds 1 + 2 to equal 3', () => {
expect(add(1, 2)).toBe(3);
});

Output: Test result, either passed or failed.

JavaScript Frameworks

JavaScript frameworks are libraries that help developers build complex web applications with a structure that is easier to maintain and scale. Subtopics:
  • React.js: A popular library for building user interfaces, particularly for single-page applications.
  • Vue.js: A progressive framework for building user interfaces, offering flexibility and ease of integration.
  • Angular: A platform for building mobile and desktop web applications using TypeScript and HTML.
  • Node.js: A runtime environment that allows you to run JavaScript on the server side.
  • Express.js: A minimal and flexible Node.js web application framework for building APIs and web applications.
  • Svelte: A framework that shifts much of the work to compile time, producing highly optimized vanilla JavaScript.
  • Backbone.js: A lightweight framework providing structure to web applications by providing models with key-value binding and custom events.

Browser APIs

Browser APIs allow web applications to interact with the browser and provide various functionalities such as storage, networking, and media playback. Subtopics:
  • DOM Manipulation: APIs that allow dynamic changes to the document structure, style, and content.
  • Fetch API: A modern way to make network requests to servers, providing a more powerful and flexible feature set than XMLHttpRequest.
  • Web Storage API: Provides storage capabilities for web applications, including local storage and session storage.
  • Canvas API: Allows for dynamic, scriptable rendering of 2D shapes and bitmap images.
  • WebSockets: Provides a way to open a persistent connection between the client and server for real-time communication.
  • Geolocation API: Allows web applications to access the user's geographical location.
  • Service Workers: Scripts that run in the background, separate from a web page, providing functionalities such as offline access and push notifications.
  • Notification API: Allows web pages to send notifications to users even when the page is not active.

JavaScript Basics

JavaScript basics cover the foundational concepts that every JavaScript developer should know before moving on to more advanced topics. Subtopics:
  • Variables and Data Types: Understanding how to store data using variables and the types of data available in JavaScript.
  • Operators: The various operators used in JavaScript for arithmetic, comparison, and logical operations.
  • Control Structures: Constructs like if statements, switch statements, and loops that control the flow of execution.
  • Functions: Reusable blocks of code that perform specific tasks and can be called upon as needed.
  • Events: Understanding how to handle user interactions with the application through events.
  • Scope: The visibility of variables in different parts of your code, including global and local scopes.
  • Arrays and Objects: Data structures used for storing collections of data and key-value pairs, respectively.

Security in JavaScript

Security in JavaScript is crucial for protecting web applications from vulnerabilities and attacks. Subtopics:
  • Cross-Site Scripting (XSS): A security vulnerability that allows attackers to inject malicious scripts into web pages viewed by users.
  • Cross-Site Request Forgery (CSRF): An attack that tricks the user into submitting a request that they did not intend to make.
  • Content Security Policy (CSP): A security measure that helps prevent XSS attacks by controlling the sources of content that can be loaded on a page.
  • Data Validation: The process of ensuring that user input is valid and safe before processing it.
  • HTTPS and Secure Connections: The importance of using secure protocols for communication between clients and servers.
  • Cookie Security: Best practices for using cookies securely, including HttpOnly and Secure flags.
  • Web Security Best Practices: General guidelines for writing secure code and avoiding common vulnerabilities.
